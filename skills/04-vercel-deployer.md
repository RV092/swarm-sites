# Skill 4 — Vercel Deployer

**Objectif :** Déployer les sites HTML sur Vercel gratuitement

**Outil :** Vercel CLI

---

## 🎯 Ce que ça fait

1. Initialise un projet Vercel pour chaque site
2. Pousse le HTML généré
3. Retourne une URL publique (xxx.vercel.app)
4. Site live en quelques secondes

## 🛠️ Commandes de Déploiement

### Déploiement Simple

```bash

cd output/luxe-nails-sydney
vercel --prod
```

### Déploiement Batch

```bash
# deploy-all.sh
#!/bin/bash

DEPLOY_DIR="./output"
RESULTS="./deployments.json"

echo "[" > $RESULTS

first=true
for dir in $DEPLOY_DIR/*/; do
  if [ -d "$dir" ]; then
    cd "$dir"
    
    # Deploy
    URL=$(vercel --prod --yes 2>&1 | grep -o 'https://[^"]*')
    
    # Log result
    if [ "$first" = true ]; then
      first=false
    else
      echo "," >> $RESULTS
    fi
    
    echo "{\"folder\": \"$(basename $dir)\", \"url\": \"$URL\"}" >> $RESULTS
    
    cd - > /dev/null
  fi
done

echo "]" >> $RESULTS

echo "✅ All deployments complete!"
cat $RESULTS
```

## 📁 Structure pour Vercel

Chaque dossier doit contenir :

```
luxe-nails-sydney/
├── index.html        # Obligatoire
├── vercel.json       # Optionnel (config)
└── assets/           # Images, CSS
    └── images/
```

### vercel.json (optionnel)

```json
{
  "version": 2,
  "builds": [
    { "src": "**/*.html", "use": "@vercel/static" }
  ],
  "routes": [
    { "src": "/(.*)", "dest": "/$1" }
  ]
}
```

## 🎯 Processus Agent

### Étape 1 — Vérifier structure

```javascript
// verify-structure.js
const fs = require('fs');

function verifyStructure(dir) {
  const indexPath = `${dir}/index.html`;
  
  if (!fs.existsSync(indexPath)) {
    throw new Error(`No index.html in ${dir}`);
  }
  
  // Verify HTML is valid
  const html = fs.readFileSync(indexPath, 'utf8');
  if (!html.includes('<!DOCTYPE html>')) {
    throw new Error(`Invalid HTML in ${dir}`);
  }
  
  return true;
}
```

### Étape 2 — Déployer

```javascript
// deploy.js
const { execSync } = require('exec-sync');

async function deploy(dir) {
  const originalDir = process.cwd();
  
  try {
    process.chdir(dir);
    
    const output = execSync('vercel --prod --yes 2>&1');
    const urlMatch = output.match(/https:\/\/[^\s]+\.vercel\.app/);
    
    return {
      success: !!urlMatch,
      url: urlMatch ? urlMatch[0] : null
    };
  } finally {
    process.chdir(originalDir);
  }
}
```

// generate-report.js

## ⚠️ Points d'Attention

- **Rate limiting** : Vercel limite les déploiements simultanés
- **Noms uniques** : Chaque projet doit avoir un nom unique
- **Validation** : Toujours vérifier que le HTML est valide avant déploy
- **Cleanup** : Supprimer les anciens déploiements pour éviter l'encombrement
